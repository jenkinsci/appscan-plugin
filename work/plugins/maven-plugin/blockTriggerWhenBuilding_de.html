Während ein neuer Build eines vorgelagerten Projeks gebaut wird (oder in der Warteschlange ist), wird ein älterer Build standardmäßig keine Builds von nachgelagerten Modulen auslösen. Dem liegt die Annahme zugrunde, dass solche Builds bald veraltet und somit uninteressant sein werden.

Sie können diese Logik aber im vorgelagerten Projekt deaktivieren und so eine vollständige Abfolge von Builds nachgelagerter Projekte erreichen. Dies ist nicht mit der Option Build blockieren, solange vorgelagertes Projekt gebaut wird des anderen Projekts zu verwechseln, welche den Ausführungszeitpunkt des nachgelagerten Projekts steuert, nachdem dieses in die Warteschlange gestellt wurde.
